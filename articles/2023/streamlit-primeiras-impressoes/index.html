Aplicações web para projetos de Data Science e IA com Streamlit

  • seg 26 junho 2023
  • utils
  • Giliard Godoi
  • #streamlit

Este artigo não é um tutorial de como criar uma aplicação com Streamlit, porém é um relato da minha experiência utilizando o Streamlit Community.

Streamlit é uma biblioteca low code que auxilia a construir aplicações sofisticadas para projetos de Data Science e Inteligência Artificial. Imagine que você quer prototipar rapidamente uma tela para o seu cliente, para demonstrar o predição de seu modelo, ou como ele poderá ser utilizado pelos consultores internos da empresa. São nesses casos que o streamlit pode ser utilizado.

Fácil desde a instalação (tudo o que você precisa é fazer pip install streamlit) uma aplicação é definida por uma script em Python. Por exemplo para criar uma tela de boas vindas, só precisaríamos do seguinte script:

import streamlit as st

'''# Boas Vindas ao mundo do Streamlit'''

nome = st.text_input('Digite o seu nome:')

st.write(f'Seja bem vindo, {nome}')

Para executar esse código e exibir no seu navegador, basta executar o comando streamlit run [app.py](http://app.py) ( digamos que o script foi salvo no arquivo app.py).

Observe que a mensagem de boas vindas foi escrita como uma string no corpo do cript,sem definir uma variável, e também utilizamos a função st.write para escrever uma mensagem customizada. O streamit entende que as duas formas são mensagem que devem ser exibidas ao usuário.

O streamlit possui uma documentação relativamente curta, fácil de entender, o que permitiu que em poucas semanas eu dominasse a maior partes das eaturesda biblioteca. Existem diversos componentes prontos que podem ser invocados como mostrado no exemplo anterior como st.text_input. Além do text_input, existem widgets como botões, checkboxes, seletores, seletores com valores múltiplos, sliders. Também existem componentes que permitem o upload de arquivos para aplicação ou entrada de imagens pela câmera do computador. Vale a pena dar uma conferida na documentação do *streamlit.*

Para usuários avançados existe a possibilidade de criar componentes customizados, o que é ensinado uma playlist de tutoriais feitos por desenvolvedores da biblioteca. Por debaixo dos panos, os componentes são definidos utilizando React e TypeScript. O que parece ser bem divertido, para quem gosta. Porém, vários componentes desenvolvidos por terceiros, de maneira não oficial, são disponibilizados como pacotes. Alguns exemplos podem até mesmo ser encontrados no site do streamlit.

Como uma aplicação é definida em um script, toda vez que ocorre uma interação na tela, como o clique em um botão, todo o script é executado novamente.

E isso é um fato importante, e alguns cuidados devem ser tomados para não se concluir com uma interface lenta toda vez que ocorre uma interação.

Alguns desses cuidados são:

  1. Utilizar os recursos de caching (ver documentação) nas operações de leitura de dados, estabelecer conexões com banco de dados, ou subir um modelo de predição.
  2. Comportamentos estranhos devido a constante atualização do estado (valores) das variáveis e componentes. (Veja essa postagem para entender mais sobre esse caso).

O desenvolvimento do streamlit é frenético, com uma nova versão saindo a cada mês. Se por um lado isso adiciona novas e interessantes eaturesà biblioteca, por outro, algumas APIs podem quebrar de uma versão para outra em pouco tempo. Existem até mesmo algumas funções teste que são precedidas pela palavra experimental para justamente indicar que podem haver mudanças no futuro, sem aviso prévio. (Isso é detalhado nessa seção).

A empresa mantenedora do projeto também oferece o Streamlit Community que permite fazer o deploy de aplicações na nuvem.

O código da aplicação precisa estar em um repositório do GitHub e após permitir que o Streamlit Community se conecte a sua conta no Git Hub, os passos para subir o aplicativo são bem simples, basta seguir o tutorial desse post aqui.

A versão gratuita do Streamlit Community permite aplicações que consuma até 1Gb de memória, o que eu acho pouco, se pensarmos que alguns modelos de Deep Learning podem ser grandes. Além disso, a versão gratuita permite apenas um app privado, e promete uma quantidade ilimitada de apps públicos, onde qualquer pessoa na web pode pesquisar e acessar.

Recentemente eu fiz um teste com o Streamlit Community, com uma aplicação que consulta uma API de documentos (o famoso SAPL) e retorna um relatório com documentos cadastrados em determinada semana. O repositório dessa aplicação pode ser encontrado nesse link.
